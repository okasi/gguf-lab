When running large language models locally, achieving high tokens per second relies heavily on optimizing three distinct hardware and algorithmic bottlenecks: quantization, memory bandwidth, and speculative decoding. Quantization reduces the precision of model weights, typically from 16-bit floating point to 4-bit or 8-bit integers. This compression directly decreases the amount of data that must be transferred between the GPU’s global memory and its processing cores. Since local inference is often memory-bound rather than compute-bound, smaller model sizes allow for faster data retrieval, significantly boosting throughput without requiring more powerful hardware.

Memory bandwidth acts as the primary physical constraint in this process. During generation, the model must load weights and access the growing context window for every new token. If the bandwidth is saturated, the processing cores sit idle, waiting for data. Therefore, maximizing tokens per second often depends on ensuring the model fits comfortably within the available VRAM, allowing the memory bus to operate at peak efficiency. Higher bandwidth directly correlates with faster token generation, making it a critical metric for local setups.
