Achieving high tokens per second in local Large Language Models requires optimizing three distinct computational bottlenecks: precision, data movement, and generation efficiency. Quantization reduces model precision from standard 16-bit floating point to lower formats like 4-bit or 8-bit integers. This compression significantly shrinks the model’s footprint, allowing larger architectures to fit within limited GPU or CPU VRAM. More critically, it reduces the amount of data the processor must read during inference, directly alleviating pressure on memory bandwidth. Since local inference is predominantly memory-bound rather than compute-bound, the speed at which weights are transferred from RAM to the processing unit dictates throughput. Higher memory bandwidth allows the hardware to fetch these compressed weights faster, resulting in a linear increase in generation speed. Therefore, quantization acts as a lever to maximize the utility of available bandwidth.

Speculative decoding offers a different approach by reducing the number of forward passes required for generation. It employs a smaller, faster "draft" model to propose multiple subsequent tokens simultaneously. The larger, primary model then verifies these proposals in a single pass. If the draft is accurate, the large model accepts the tokens without individual processing, effectively skipping steps. This method dramatically lowers the computational load per accepted token, increasing throughput without requiring additional hardware resources
