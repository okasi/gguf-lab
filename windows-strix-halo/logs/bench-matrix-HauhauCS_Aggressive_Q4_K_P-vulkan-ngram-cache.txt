Running large language models locally requires balancing three critical hardware constraints that directly dictate token generation speed. Quantization reduces model precision, typically from 16-bit floating point to 4-bit or 8-bit integers, which significantly shrinks the model’s footprint in RAM. This reduction allows larger models to fit into consumer-grade memory, but more importantly, it decreases the data volume the CPU or GPU must process per calculation. Consequently, lower precision often accelerates inference by reducing arithmetic complexity, though excessive compression can introduce latency if the decompression overhead outweighs the computational savings.

Memory bandwidth serves as the primary bottleneck for autoregressive generation. During text completion, the model must read the entire context window and the model weights from memory for every single token produced. Since modern GPUs often compute faster than memory can supply data, high bandwidth is essential for sustaining high tokens per second. If the memory bus is saturated, the compute units sit idle, waiting for data, which drastically lowers throughput regardless of the processor’s raw power.
