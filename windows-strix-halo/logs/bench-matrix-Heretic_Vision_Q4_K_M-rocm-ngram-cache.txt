Quantization, memory bandwidth, and speculative decoding are the three primary levers controlling token generation speed in local large language models. Quantization reduces the precision of model weights, typically from 16-bit floating point to 4-bit or 8-bit integers. This compression directly lowers the memory footprint, allowing larger models to fit into limited VRAM. More critically, it reduces the amount of data the GPU must read for every inference step. Since local LLM inference is predominantly memory-bound rather than compute-bound, the speed at which weights are transferred from VRAM to the processing cores dictates throughput. Memory bandwidth represents this physical transfer limit. Higher bandwidth allows more data to flow per second, directly increasing tokens per second. If your model is too large for your GPU’s memory capacity, the system offloads layers to system RAM, causing severe latency spikes due to significantly slower data transfer rates. Therefore, keeping the entire model within high-speed VRAM is essential for maintaining consistent speed. Speculative decoding offers a computational shortcut to bypass some memory bottlenecks. It employs a smaller, faster "draft" model to predict multiple future tokens simultaneously. The larger target model then verifies these predictions in parallel rather than sequentially.
