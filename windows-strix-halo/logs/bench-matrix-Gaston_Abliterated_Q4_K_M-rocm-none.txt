Local large language model performance, specifically measured in tokens per second, is primarily constrained by hardware limitations rather than raw computational power. Quantization serves as the first major lever for optimization by reducing the precision of model weights from standard 16-bit floating point to lower-bit integers like 4-bit or 8-bit. This reduction drastically shrinks the model‚Äôs memory foot print, allowing larger models to fit within the limited VRAM of consumer GPUs. More importantly, smaller data types require significantly less energy and time to transfer between memory and the processing cores, directly accelerating the speed at which the model can ingest and process data during generation.

However, the dominant bottleneck for local LLMs is memory bandwidth, often referred to as being memory-bound. Unlike training, which is compute-bound, autore generation is limited by how fast the GPU can read the entire model from VRAM for every single token generated. Since the model must be fully loaded into memory for each step, the throughput is strictly capped by the memory bus width and speed. High-bandwidth memory architectures are therefore essential; without sufficient bandwidth, even the most powerful GPU cores sit idle, waiting for data. Consequently, maximizing tokens per second requires minimizing the amount of data moved, which is where quantization excels.
