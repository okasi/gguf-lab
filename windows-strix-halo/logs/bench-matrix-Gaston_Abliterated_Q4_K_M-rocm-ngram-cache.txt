Local Large Language Model performance, measured in tokens per second, is primarily constrained by hardware limitations and optimized through sophisticated algorithmic techniques. The most significant bottleneck is typically memory bandwidth, often referred to as the "memory wall." Since LLMs are memory-bound rather than compute-bound during generation, the speed at which weights can be transferred from VRAM to the processing units dictates throughput. High-bandwidth memory (HBM) is therefore critical; if the data transfer rate is slow, the GPU cores sit idle, drastically reducing token generation speed regardless of computational power.

To mitigate this, quantization reduces the precision of model weights, such as moving from 16-bit floating-point to 4-bit or even lower. This compression shrinks the model’s memory footprints significantly, allowing more of the network to fit within the limited VRAM capacity. Crucially, smaller weights require less data to be moved per token, effectively increasing the effective bandwidth utilization and enabling faster generation without requiring expensive hardware upgrades. However, aggressive quantization can introduce latency artifacts or quality degradation if not implemented carefully.
